Tool calling — sometimes called function calling — is when you give an LLM a list of tools it can use, and the model decides when and how to call them. Instead of just generating text, the model can now take actions: search the web, query a database, send an email, run code, or call any API you define. Think of it this way. A regular chatbot is like a person who can only talk. An agent with tool calling is like a person who can talk AND use a computer. That's a massive difference. The model doesn't actually execute the tools — it outputs a structured request saying "I want to call this function with these arguments." Your code executes the function, returns the result, and the model uses that result to continue reasoning. It's a loop: think, act, observe, repeat. Every major model now supports tool calling, but the implementations vary. OpenAI's GPT-5 and o3 use a tools parameter in the API. You define your tools as JSON schemas, and the model returns tool_calls in its response. It's clean, well-documented, and the most widely adopted format. Anthropic's Claude takes a similar approach but adds some nice touches. Claude's tool use is particularly strong at multi-step reasoning — deciding which tools to call in what order. That's what makes agents fundamentally different from traditional chatbots. A few things I've learned building agents in production. First — keep your tool descriptions crystal clear. The model decides which tool to call based on the description. Vague descriptions lead to wrong tool choices. Be specific about what each tool does and when it should be used. Second — handle errors gracefully. Tools fail. APIs time out. Return clear error messages so the model can adapt. A good agent will retry, try a different approach, or tell the user what went wrong. Third — set limits on the loop. An agent that can call tools indefinitely will occasionally get stuck in loops. Set a maximum number of tool calls per turn — ten is a reasonable default. Fourth — log everything. Agent behavior is non-deterministic. You need to see exactly which tools were called, with what arguments, and what results came back.
